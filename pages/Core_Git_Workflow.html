Core Git Workflow

The Essential Steps of Working with Git

Rimas Alharbi

1. Check Status

  • Shows the current state of your project
  • Detects new, modified, or deleted files
  • Helps you understand what needs to be added or committed
git status

2. Add Files

  • Moves files to the Staging Area
  • Prepares files for the next commit
git add .
git add file.html

3. Commit Changes

  • Saves a snapshot of your project
  • Requires a clear and meaningful message
git commit -m "Message"

4. Push to GitHub

  • Uploads your commits to GitHub
  • Makes your work available to your team
git push

Commit Message Tips

Be clear and specific Describe what changed and why Avoid vague messages

Bad Commit Messages

  • "update"
  • "fix"
  • "final version"
  • "changes"

Good Commit Messages

  • Add navigation bar to homepage
  • Fix login validation bug
  • Update README with setup steps
  • Refactor CSS for readability
