Creating a GitHub Repository

From new repo to README and .gitignore

Khawlah Almansouri

Creating a New Repository

  1. اضغطي على زر + في أعلى يمين GitHub.
  2. اختاري New repository.
  3. اكتبي اسم المستودع (Repository name).
  4. اكتبي وصف بسيط للمشروع (اختياري).
  5. اختاري الإعدادات ثم اضغطي Create repository.

Public vs Private

  • Public: الكل يقدر يشوف الكود، مناسب للمشاريع التعليمية أو open-source.
  • Private: بس إنتِ والأشخاص اللي تعطيهم صلاحية يقدروا يشوفوا الكود.
  • تقدرين تغيّري نوع الرپوزيتوري لاحقاً من الإعدادات.

README.md

  • هو ملف يشرح فكرة المشروع وكيفية تشغيله.
  • يُكتب بلغة Markdown مثل العناوين، القوائم، والروابط.
  • يفضّل اختيار خيار Initialize with a README عند إنشاء الرپوزيتوري.
# Project Title
Short description of your project.

## Features
- Feature 1
- Feature 2

## How to run
Steps to run the project.

.gitignore

  • ملف يحدد الملفات والمجلدات اللي Git يتجاهلها (ما يرفعها على GitHub).
  • مفيد لتجاهل ملفات النظام، ملفات الـ IDE، أو الملفات السرية مثل كلمات المرور.
  • تقدرين تختارين قالب جاهز من GitHub حسب نوع المشروع (Node, Python, etc.).
# Example .gitignore
node_modules/
.env
.vscode/
*.log

Repository Naming Conventions

  • خلي الاسم واضح ويعبر عن المشروع، مثل: portfolio-website أو todo-app.
  • تجنبي المسافات، واستخدمي الشرطات - بين الكلمات.
  • استخدمي حروف صغيرة لتكون الأسماء أسهل في القراءة والمشاركة.
Good: ecommerce-product-catalog
Bad: project123
